Proportions & Specs

Body Radius: rx: 0.36 unit
Point: triangle, tip 0.9 unit below body
Legs: 2 lines, stroke-width: 4px, linecap: round
Leg Splay: 3px outward from body edges
Head Gap: 6px between head bottom and body top

Scale Rule: All proportions are relative. When resizing, scale the entire character uniformly. The head:body ratio must remain constant. Minimum display size: 24px total height.

07

Usage Guidelines

✓ Do

ALWAYS use animated V-bounce head version in apps, websites, splash screens. The moving head IS the logo identity.

✗ Don't

Don't use static logo in digital where animation is possible. Static is ONLY for print, favicon, and non-animated contexts.

✓ Do

Use logo with tight spacing. Character sits snugly between th and ya.

✗ Don't

Don't add extra spacing around the character. No wide gaps.

✓ Do

Scale entire logo uniformly. Head:body ratio must stay constant at all sizes.

✗ Don't

Don't stretch, skew, or change character proportions independently.

✓ Do

Use #FF6B35 character on dark bg. Use white character on orange bg.

✗ Don't

Don't change character color to blue, green, or any non-brand color.

✓ Do

Minimum clear space around logo = height of character head on all sides.

✗ Don't

Don't place logo on busy backgrounds or overlap with other elements.

★ For Developers — CRITICAL: The V-bounce head animation is NOT optional — it's the primary brand identity. Implement it on every screen where the logo appears. All animations are CSS-only — no JS runtime needed. For Lottie export, recreate these timing curves in After Effects. For Android/iOS native, use spring-based interpolators for bounce-squash and ease-in-out curves for everything else. V-bounce head + walking feet = non-negotiable in digital.
