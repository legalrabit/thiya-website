KYC — API Integration Guide

Complete endpoint reference for mobile app KYC implementation. Step 4 of vendor onboarding.

Key Rules PAN and Aadhaar are mandatory. Bank, FSSAI, GST are optional — vendor can set up later from profile. Payment features unlock only after KYC approval.

Upload Flow

Two-step process per document: upload the file first, then create the document record.

1 Upload File POST /files/upload
→
2 Create Document POST /vendor/documents/my
→
3 Admin Reviews PENDING → APPROVED

Vendor Endpoints

1. Upload File

POST /api/v1/files/upload
Upload document image/PDF. Returns fileId for use in step 2.
Content-Type: multipart/form-data
FieldTypeRequiredDescription
fileFileYesJPG, PNG or PDF. Max 5MB
categoryStringYesKYC_AADHAAR, KYC_PAN, KYC_GST, KYC_LICENSE, KYC_OTHER
ownerTypeStringYesVENDOR
ownerIdUUIDYesVendor ID

Response:

{
  "success": true,
  "data": {
    "id": "a1b2c3d4-...",
    "originalName": "aadhaar_front.jpg",
    "contentType": "image/jpeg",
    "size": 245000,
    "url": ".../download",
    "thumbnailUrl": ".../thumbnail"
  }
}

2. Create KYC Document

POST /api/v1/vendor/documents/my
Creates a document record linked to the uploaded file.
Permission: vendor:kyc:write

Body:

{
  "documentType": "AADHAAR",
  "fileId": "a1b2c3d4-...",
  "documentNumberMasked": "****1234",
  "nameOnDocument": "Abhash Kumar"
}

Document Types:

ValueRequiredDescription
AADHAARYesAadhaar card (12-digit ID)
PANYesPAN card (ABCDE1234F)
BANK_ACCOUNTNoBank passbook / cancelled cheque
FSSAINoFood license (food vendors only)
GSTNoGST certificate

3. List My Documents

GET /api/v1/vendor/documents/my
Returns all uploaded documents with their review status.
Permission: vendor:kyc:read
{
  "success": true,
  "data": [
    {
      "id": "doc-uuid-1",
      "documentType": "AADHAAR",
      "fileId": "file-uuid-1",
      "documentNumberMasked": "****1234",
      "reviewStatus": "PENDING",
      "createdAt": "2026-04-06T10:30:00Z"
    },
    {
      "id": "doc-uuid-2",
      "documentType": "PAN",
      "reviewStatus": "APPROVED",
      "reviewedAt": "2026-04-06T11:00:00Z"
    }
  ]
}

4. Resubmit Rejected Document

PATCH /api/v1/vendor/documents/my/{documentId}/resubmit?newFileId={uuid}
Re-upload a rejected document with a new file. Resets status to PENDING.
Permission: vendor:kyc:write

5. View / Download File

GET /api/v1/files/{fileId}/download
Returns file binary with Content-Type header.
GET /api/v1/files/{fileId}/thumbnail
Returns 200x200 thumbnail (images only).

Admin Endpoints

Used by admin web portal for document review.

GET /api/v1/vendor/documents/pending
List all documents awaiting review.
Permission: vendor:kyc:review
PATCH /api/v1/vendor/documents/{id}/approve
Approve a document.
PATCH /api/v1/vendor/documents/{id}/reject?reason=text
Reject a document with reason. Vendor can resubmit.

Document Status Flow

PENDING → APPROVED
or
PENDING → REJECTED → resubmit → PENDING

Vendor KYC Status (Aggregate)

The vendor's overall kycStatus on their profile reflects document states:

StatusMeaning
NOT_APPLIEDNo documents uploaded yet
PENDINGAt least one document uploaded, awaiting review
APPROVEDAll required documents (PAN + Aadhaar) approved
REJECTEDOne or more required documents rejected

Check via: GET /api/v1/vendor/profile/me → kycStatus field

Mobile Implementation Notes

Upload Flow (per document)

  1. User taps document card
  2. Open camera / file picker
  3. POST /files/upload (multipart) → get fileId
  4. POST /vendor/documents/my with { documentType, fileId, documentNumberMasked }
  5. Show success state on card
  6. Repeat for next document

Mandatory Check

Submit Button Logic Enable "Submit" only when both AADHAAR and PAN are uploaded. Other documents (BANK, FSSAI, GST) are optional and don't block onboarding.

Error Handling

ErrorAction
File too large (>5MB)Show "File too large. Please compress or crop."
Duplicate document typeCall resubmit endpoint instead of create
Network error during uploadRetry with exponential backoff
Rejected documentShow rejection reason, allow re-upload

WhatsApp Notifications

  • After submit: "Documents received, under review"
  • On approval: "KYC approved! Payments unlocked."
  • On rejection: "Document rejected: {reason}. Please re-upload."

cURL Examples

Upload file

curl -X POST http://localhost:8080/api/v1/files/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@aadhaar.jpg" \
  -F "category=KYC_AADHAAR" \
  -F "ownerType=VENDOR" \
  -F "ownerId=<vendor-uuid>"

Create document

curl -X POST http://localhost:8080/api/v1/vendor/documents/my \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "documentType": "AADHAAR",
    "fileId": "<file-uuid>",
    "documentNumberMasked": "****1234",
    "nameOnDocument": "Abhash Kumar"
  }'

List my documents

curl http://localhost:8080/api/v1/vendor/documents/my \
  -H "Authorization: Bearer <token>"
